# ICCS 102 – Live Debugging Challenge v1.0

## Assignment Type

In-class team activity

Temporary teams of **3 students**

## Learning Objective

By completing this activity, students should be able to:

* identify common beginner Python mistakes,
* explain why code does not run correctly or does not behave correctly,
* correct syntax errors, runtime errors, and simple logic errors,
* apply variables, input/output, type conversion, operators, conditions, loops, and lists,
* trace code step by step,
* communicate debugging ideas clearly with teammates.

## Duration

Recommended total class time: **1–2 class sessions**

The instructor may use as many problems as time allows.

## Task Description

The instructor will show broken Python code snippets on the projector.

Each code snippet contains **one or two mistakes**.

Your team must:

1. identify the mistake,
2. explain why it is wrong,
3. suggest a correct fix,
4. observe the code being run again after correction.

The goal is not only to guess the answer. The goal is to understand **why** the mistake happens.

## Team Rules

* Work in temporary teams of 3.
* Every team should discuss every problem.
* Students may **not** open an IDE, run the code themselves, use AI tools, or use Google to analyze the code before answering.
* Students may only look at the projected code, discuss with teammates, and then participate in class.
* The instructor will choose which team answers first for each round.
* If the first team is incorrect or incomplete, another team may answer.
* Any team member may be asked to explain the bug or the fix.
* Be respectful when another team is answering.
* The instructor has the final say on whether an answer is correct.
* Failure to follow the no-IDE / no-AI / no-Google rule may result in a score of **0 for the whole group** for this activity.

## Recommended Round Format

For each problem:

1. (1 min ) The whole class reads the code silently.
2. (2 mins) Teams discuss the possible mistake and fix.
3. (2 mins) Instructor chooses arbitarily one team answers first. (Chosen team is rotated)
4. (1 min ) If needed, another team may answer.
5. (1 min ) The instructor confirms the mistake.
6. (1 min ) The corrected code is run again on the projector.

Total round time: 6-10 mins

## Suggested Scoring

| Action                            |          Points |
| --------------------------------- | --------------: |
| Correctly identify the bug        | +0.5 team point |
| Correctly explain why it is wrong | +0.5 team point |
| Correctly suggest the fix         | +0.5 team point |

The instructor may adjust scoring as needed.

## Topics Covered

The snippets may include mistakes involving:

* variable names,
* naming rules,
* data types,
* `input()` and type conversion,
* arithmetic operators,
* comparison operators,
* boolean logic,
* `if / elif / else`,
* indentation,
* `for` loops,
* `while` loops,
* infinite loops,
* `break` and `continue`,
* lists,
* `append()`,
* list indexing,
* loop tracing,
* common beginner debugging mistakes.

## Submission Format

No formal student submission is required unless the instructor says otherwise.
